# Mobile App Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the mobile PDF gate — an Expo + React Native + TypeScript app with a Home screen holding navigation buttons, a **Tareas** section backed by Redux (list + add-via-modal + empty-validation + persistence across navigation), and a **Listado** section that fetches the mandated mockapi source on mount with a loading layout — every screen and the redux slice covered by Jest unit tests. Then layer three additive, gracefully-degrading bonuses (redux-persist, a backend-served **Marcas** screen, and a tasks **Sync** button) without touching any required behavior.

**Architecture:** Single Expo app under `mobile/`. React Navigation **native-stack** owns screen routing; screens are plain components tested in isolation with a mock `navigation` prop. **Redux Toolkit** is the single source of truth for Tareas (`tasks` slice), consumed through typed hooks. Remote data lives behind thin `src/api/*` fetch clients (one per source) so screens depend on a function, not on `fetch` — which makes screen tests trivial to mock. The mandated mockapi and the (bonus) backend are separate clients; **Tareas and Listado never depend on the backend**.

**Tech Stack:** Expo (latest SDK) + React Native + TypeScript, React Navigation (`@react-navigation/native` + `native-stack`), Redux Toolkit + react-redux, redux-persist + AsyncStorage (bonus), Jest with the `jest-expo` preset + `@testing-library/react-native`.

## Global Constraints

- **Expo managed workflow**, **TypeScript** everywhere, `strict` mode on (from the `blank-typescript` template). The app runs on **Android** (`expo start --android`); unit tests run in **Node** with no emulator.
- **Navigation:** React Navigation **native-stack** only. Screens receive `navigation`/`route`; no `useNavigation` inside screen bodies (keeps unit tests prop-driven).
- **State:** Redux Toolkit (`configureStore`, `createSlice`). No legacy Redux boilerplate. Tareas state is the **source of truth** and must survive navigating away and back (in-memory Redux satisfies this; redux-persist is a bonus for surviving app restart).
- **Empty-task rule (PDF):** creating a task with a blank/whitespace-only description is rejected — enforced in BOTH the modal UI (submit disabled + guarded) and the reducer (no-op), so it is unit-testable at the slice level.
- **Listado source is EXACTLY** `https://6172cfe5110a740017222e2b.mockapi.io/elements` — fetched **on mount**, with a **loading layout** while pending, rendering at least `name` (optional `avatar` contact-list style). The endpoint returns objects shaped `{ id: string, name: string, avatar: string, createdAt: string }`.
- **Testing (PDF):** every screen AND the redux slice have Jest unit tests (`jest-expo` + `@testing-library/react-native`). Remote calls are mocked; no network in tests. Target line coverage **>= 80%**.
- **Testing conventions (RTL v14 — verified against this exact toolchain, non-negotiable):** `@testing-library/react-native` v14's `render`, `fireEvent`, and `unmount()` are **all async** — always `await render(...)`, `await fireEvent.press(...)` / `await fireEvent.changeText(...)`, `await result.unmount()`, and `renderWithProviders` is an **async** helper you must `await`. Prefer global `screen` queries (`import { screen } from '@testing-library/react-native'`). After `fireEvent`, the re-render is **NOT synchronous**: assert rendered output with `await screen.findByText(...)` (or `await waitFor(...)`), never a bare synchronous `getByText`. The pressed handler runs synchronously, so a Redux `dispatch` inside it lands immediately — assert store logic via `store.getState()` right after `fireEvent`. **Any test that renders a `FlatList` must `await flushListTimers()` (a `test-utils` helper wrapping a ~60ms `setTimeout` in `act`) at the end**, or `VirtualizedList` emits `"not wrapped in act(...)"` warnings. Test output MUST be pristine (zero `console.error`/warnings). `mobile/tsconfig.json` sets `"types": ["jest"]`; `mobile/package.json` `jest.moduleNameMapper` maps `immer` AND `react-redux` to their CJS builds (jest-expo resolves both to untransformed ESM otherwise). (Install order: `npm install` before any `npx expo install` so `npx` never prompts; run installs in the foreground.)
- **UI copy language:** the product identity is Spanish ("Tareas y gestión de catálogos") and the challenge screen names are Spanish, so **user-facing strings are in neutral Spanish** (button titles, loading/empty/error text). **All identifiers, types, comments, and file names are in English.** Established Spanish domain names stay Spanish (`Tareas`, `Listado`, `Marcas`, `descripcion`, `nombre`, `paisOrigen`).
- **Standalone-first:** the Tareas and Listado features MUST work with the backend completely offline/absent. The bonus features (**Marcas**, **Sync**) are additive, opt-in, self-announcing, and **degrade gracefully** (friendly state, never a crash) when the backend is unreachable.
- **Backend base URL** is configurable via `process.env.EXPO_PUBLIC_BACKEND_URL`, defaulting to `http://localhost:8080`. Backend JSON is camelCase: marcas `{ id, nombre, paisOrigen, fechaCreacion }`; sync request `{ descripciones: string[] }` → response `{ imported, skipped, tasks: { id, descripcion, fechaCreacion }[] }`.
- Commits follow **Conventional Commits** with the `mobile` scope (`feat(mobile): ...`, `test(mobile): ...`, `chore(mobile): ...`). **No AI attribution / Co-Authored-By lines.**
- Repository root is `d:/Julio/Git/coderland-challenge-backend`. The mobile app lives under `mobile/`. Work happens on branch `feat/mobile`.

Create `mobile/src/store/tasksSlice.ts`:

```ts
import { createSlice, nanoid, type PayloadAction } from '@reduxjs/toolkit';
import type { Task } from '../types/task';
import type { RootState } from './index';

interface TasksState {
  items: Task[];
}

const initialState: TasksState = { items: [] };

const tasksSlice = createSlice({
  name: 'tasks',
  initialState,
  reducers: {
    addTask: {
      reducer(state, action: PayloadAction<Task>) {
        if (!action.payload.description) {
          return; // guard: empty descriptions are rejected
        }
        state.items.push(action.payload);
      },
      prepare(description: string) {
        return { payload: { id: nanoid(), description: description.trim() } };
      },
    },
  },
});

export const { addTask } = tasksSlice.actions;
export default tasksSlice.reducer;

export const selectTasks = (state: RootState): Task[] => state.tasks.items;
```

import React, { type PropsWithChildren, type ReactElement } from 'react';
import { render } from '@testing-library/react-native';
import { Provider } from 'react-redux';
import { setupStore, type AppStore, type RootState } from './src/store';

interface Options {
  preloadedState?: Partial<RootState>;
  store?: AppStore;
}

// RTL v14's render is async, so this helper is async too. Always `await renderWithProviders(...)`.
export async function renderWithProviders(
  ui: ReactElement,
  { preloadedState, store = setupStore(preloadedState) }: Options = {},
) {
  function Wrapper({ children }: PropsWithChildren) {
    return <Provider store={store}>{children}</Provider>;
  }
  return { store, ...(await render(ui, { wrapper: Wrapper })) };
}
```

import { useEffect, useState } from 'react';
import { ActivityIndicator, FlatList, StyleSheet, Text, View } from 'react-native';
import { fetchElements } from '../api/elements';
import type { Element } from '../types/element';
import PersonRow from '../components/PersonRow';

export default function ListadoScreen() {
  const [elements, setElements] = useState<Element[]>([]);

  const [error, setError] = useState(false);

  useEffect(() => {

    setError(false);

      .then((data) => {
        if (active) setElements(data);
      })
      .catch(() => {
        if (active) setError(true);
      })
      .finally(() => {
        if (active) setLoading(false);
      });
    return () => {
      active = false;
    };
  }, []);

  if (loading) {
    return (
      <View style={styles.center}>
        <ActivityIndicator size="large" />
        <Text style={styles.hint}>Cargando...</Text>
      </View>
    );
  }

  if (error) {
    return (
      <View style={styles.center}>
        <Text style={styles.hint}>No se pudo cargar el listado.</Text>
      </View>
    );
  }

  return (
    <FlatList
      data={elements}
      keyExtractor={(element) => element.id}
      renderItem={({ item }) => <PersonRow name={item.name} avatar={item.avatar} />}
    />
  );
}

import { combineReducers, configureStore } from '@reduxjs/toolkit';
import {
  persistReducer,
  persistStore,
  FLUSH,
  PAUSE,
  PERSIST,
  PURGE,
  REGISTER,
  REHYDRATE,
} from 'redux-persist';
import AsyncStorage from '@react-native-async-storage/async-storage';
import tasksReducer from './tasksSlice';

const rootReducer = combineReducers({
  tasks: tasksReducer,
});

export type RootState = ReturnType<typeof rootReducer>;

// Plain, non-persisted store used by the test factory — keeps tests isolated.
export function setupStore(preloadedState?: Partial<RootState>) {
  return configureStore({
    reducer: rootReducer,
    preloadedState,
  });
}

const persistConfig = {
  key: 'root',
  storage: AsyncStorage,
  whitelist: ['tasks'],
};

const persistedReducer = persistReducer(persistConfig, rootReducer);

export const store = configureStore({
  reducer: persistedReducer,
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware({
      serializableCheck: {
        ignoredActions: [FLUSH, REHYDRATE, PAUSE, PERSIST, PURGE, REGISTER],
      },
    }),
});

export const persistor = persistStore(store);

export type AppStore = ReturnType<typeof setupStore>;
export type AppDispatch = typeof store.dispatch;
```

Create `mobile/src/api/tasksSync.ts`:

```ts
import { BACKEND_BASE_URL } from './config';

export interface SyncedTask {
  id: number;
  descripcion: string;
  fechaCreacion: string;
}

export interface SyncResult {
  imported: number;
  skipped: number;
  tasks: SyncedTask[];
}

export async function syncTasks(descriptions: string[]): Promise<SyncResult> {
  const response = await fetch(`${BACKEND_BASE_URL}/api/tasks/sync`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ descripciones: descriptions }),
  });
  if (!response.ok) {
    throw new Error(`Sync failed: ${response.status}`);
  }
  return (await response.json()) as SyncResult;
}
```

import { useState } from 'react';
import { Button, FlatList, StyleSheet, Text, View } from 'react-native';
import { useAppSelector } from '../store/hooks';
import { selectTasks } from '../store/tasksSlice';
import { syncTasks } from '../api/tasksSync';

    } finally {

    }

      <FlatList
        style={styles.list}
        data={tasks}
        keyExtractor={(task) => task.id}
        renderItem={({ item }) => <Text style={styles.item}>{item.description}</Text>}
        ListEmptyComponent={<Text style={styles.empty}>No hay tasks todavía.</Text>}
      />
      <Button title="Sincronizar" onPress={handleSync} disabled={syncing} />
      {status ? <Text style={styles.status}>{status}</Text> : null}
      <AddTaskModal visible={modalVisible} onClose={() => setModalVisible(false)} />
    </View>
  );
