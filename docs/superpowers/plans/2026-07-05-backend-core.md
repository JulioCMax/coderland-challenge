# Backend Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the backend PDF gate — a .NET 10 API that connects to PostgreSQL via EF Core, creates and seeds a `MarcasAutos` table, exposes `GET /api/marcas`, is covered by XUnit tests at >= 70%, and runs via `docker-compose` (PostgreSQL + API).

**Architecture:** 4-layer Clean-lite (Ports & Adapters). `Domain` holds the `MarcaAuto` entity and the `IMarcaAutoRepository` port. `Application` holds the DTO + `MarcaAutoService`. `Infrastructure` holds `AppDbContext`, EF configuration, seed, and the repository adapter. `Api` holds the controller, DI composition root, OpenAPI, and health check. Dependencies point inward: `Api -> Application -> Domain`, `Infrastructure -> Domain`. `Domain` references nobody.

**Tech Stack:** .NET 10, EF Core 10 + Npgsql, XUnit + Moq + coverlet, Swashbuckle (Swagger), AspNetCore health checks, Docker Compose.

## Global Constraints

- **.NET 10** (LTS) for all projects; nullable reference types enabled.
- Database: **PostgreSQL** accessed via **EF Core** with the Npgsql provider.
- Table name is exactly **`MarcasAutos`** (PDF requirement); tables + columns use **PascalCase**.
- Seed the table with **at least 3** car brands via EF Core `HasData` (deterministic constant values only).
- Tests use **XUnit**; the test suite includes an **in-memory DB context**; total line coverage **>= 70%**.
- `docker-compose.yml` defines exactly two services: **PostgreSQL** and the **API**, and the API connects to the DB.
- All code, identifiers, comments, and docs in **English**. Commits follow **Conventional Commits**.
- Repository root is `d:/Julio/Git/coderland-challenge-backend`. The backend lives under `backend/`.

```csharp
using MarcasAutos.Domain.Entities;
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Metadata.Builders;

namespace MarcasAutos.Infrastructure.Persistence.Configurations;

public class MarcaAutoConfiguration : IEntityTypeConfiguration<MarcaAuto>
{
    // Fixed timestamp so HasData produces a deterministic migration (no DateTime.UtcNow).
    private static readonly DateTime SeedDate = new(2024, 1, 1, 0, 0, 0, DateTimeKind.Utc);

    public void Configure(EntityTypeBuilder<MarcaAuto> builder)
    {
        // PDF requirement: the table must be named "MarcasAutos".
        builder.ToTable("MarcasAutos");
        builder.HasKey(m => m.Id);

        builder.Property(m => m.Nombre).IsRequired().HasMaxLength(100);
        builder.Property(m => m.PaisOrigen).HasMaxLength(100);
        builder.Property(m => m.FechaCreacion).IsRequired();

        builder.HasIndex(m => m.Nombre).IsUnique();

        builder.HasData(
            new MarcaAuto { Id = 1, Nombre = "Toyota", PaisOrigen = "Japan", FechaCreacion = SeedDate },
            new MarcaAuto { Id = 2, Nombre = "Ford", PaisOrigen = "United States", FechaCreacion = SeedDate },
            new MarcaAuto { Id = 3, Nombre = "Volkswagen", PaisOrigen = "Germany", FechaCreacion = SeedDate });
    }
}
```

```csharp
using MarcasAutos.Infrastructure.Persistence;
using Microsoft.AspNetCore.Hosting;
using Microsoft.AspNetCore.Mvc.Testing;
using Microsoft.EntityFrameworkCore;
using Microsoft.Extensions.DependencyInjection;

namespace MarcasAutos.IntegrationTests;

/// <summary>
/// Boots the real API pipeline but swaps PostgreSQL for the EF Core in-memory
/// provider (satisfies the PDF "in-memory DB context" requirement).
/// </summary>
public class CustomWebApplicationFactory : WebApplicationFactory<Program>
{
    protected override void ConfigureWebHost(IWebHostBuilder builder)
    {
        builder.ConfigureServices(services =>
        {
            // Remove the Npgsql-backed DbContext registration.
            var descriptor = services.SingleOrDefault(
                d => d.ServiceType == typeof(DbContextOptions<AppDbContext>));
            if (descriptor is not null) services.Remove(descriptor);

            // Register the in-memory provider instead.
            services.AddDbContext<AppDbContext>(o => o.UseInMemoryDatabase("MarcasAutosTests"));

            // Create the schema + apply HasData seed for the in-memory database.
            using var scope = services.BuildServiceProvider().CreateScope();
            var db = scope.ServiceProvider.GetRequiredService<AppDbContext>();
            db.Database.EnsureCreated();
        });
    }
}
```
